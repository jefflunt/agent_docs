# 02 Patterns

This directory defines the standard engineering practices for this project.

- [ ] `coding_conventions.md`: Style, language-specific idioms, file organization.
- [ ] `testing_patterns.md`: How to write unit, integration, and end-to-end tests.
- [ ] `documentation_first.md`: Standards for treating documentation as the project contract.
- [ ] `context_management.md`: Rules for progressive disclosure and token efficiency.

*Agent Instruction*: Before contributing code, ensure your changes follow the patterns defined here. When implementing new features, prioritize updating the documentation *before* coding (Documentation-First) so that the documentation is used as a design guideline.
